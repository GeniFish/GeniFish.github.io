为什么用let而不再用var

发表于 2018-07-02 | 分类于 前端学习
字数统计: 400 | 阅读时长 ≈ 2

emmm…好久没有更博客了，罪过罪过，前段时间忙项目，这段时间一大堆考试，忙的都快忘了更博客这回事了，还好事情差不多终于都搞定了，又可以开心地搞前端啦♪(^∇^*)

今天就记录一个简单的但是缺很重要的变化吧，在ES6出现之前，大家声明变量都是用的var，但是任何使用var声明的变量都会被提升至作用域的顶部

//你的代码
x;        //  undefined
var x = 3;
x;        //  3

呐呐呐，用var声明的变量，竟然可以在声明前被引用，这是因为JavaScrip会将上面的代码翻译成下面的形式：

var x;  // 声明
x;        // undefied
x = 3;
x;        // 3

使用let时，如果引用的变量未声明，将会得到一个错误。

x;        // ReferenceError: x 未定义
let x = 3;        // 不会执行，因为前面发生错误终止了程序

另外使用let可以起到变量屏蔽的作用，废话不多说，直接上代码：

{
   var x = 'blue';
   console.log(x);
   {
      var x = 3;
      console.log(x);
   }
    console.log(x);
}
    console.log(x);

用var 定义变量，输出结果是

blue
3
3
3
使用let定义就好的多，不会出现那种超出块级作用域的问题

{
   let x = 'blue';
   console.log(x);
   {
      let x = 3;
      console.log(x);
   }
    console.log(x);
}
    console.log(x);

输出结果为：

blue
3
blue
错误：ReferenceError: x is not defined

小伙伴们看到没有，var相对于let毫无优势，任何使用var的地方，都可以使用let对其进行优化，let最终肯定会完全取代var的。
