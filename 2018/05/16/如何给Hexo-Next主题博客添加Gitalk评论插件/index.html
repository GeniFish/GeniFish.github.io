如何给Hexo+Next主题博客添加Gitalk评论插件

发表于 2018-05-16
字数统计: 515 | 阅读时长 ≈ 2

记录在NexT主题中添加gitalk评论系统的步骤。

gitalk：一个基于 Github Issue 和 Preact 开发的评论插件
详情Demo可见：https://gitalk.github.io/

新建仓库

  1. Github上新建一个仓库，命名随便只要记得住就好了，例如gitalk
  2. 创建OAuth Application，没有的小伙伴戳这里去创建

参数说明：
Application name： # 应用名称，随意填，例如gitalk
Homepage URL： # 网站URL，如https://genifish.github.io
Application description： # 描述，随意
Authorization callback URL：# 网站URL，这个不能乱填！一定要填自己的博客域名

点击
Register application按钮创建后，页面跳转后，其中Client ID和Client Secret在后面的配置中需要用到，到时复制粘贴即可。

修改主题文件

在主题配置文件next/_config.yml中添加如下内容：

# Gitalk
gitalk: 
  enable: true    #用来做启用判断可以不用
  clientID: 'your clientID'    #上面生成的往这里复制粘贴
  clientSecret: 'your clientSecret'   #同上
  repo: Blog_comments    #先前创建的仓库名称，如gitalk
  owner:     #你的github用户名
  admin:     #你的github用户名
  distractionFreeMode: true

gitalk.swig

找到next/layout/_third-party/comments文件夹，新建gitalk.swig文件，代码如下：

{% if page.comments && theme.gitalk.enable %}
	  
	  
	   
	{% endif %}

comments.swig

修改/layout/_partials/comments.swig，在最后div标签中添加如下内容添加内容：

{% elseif theme.gitalk.enable %}
 <div id="gitalk-container"></div>

index.swig

修改layout/_third-party/comments/index.swig，在最后一行添加内容：

{% include 'gitalk.swig' %}

gitalk.styl

新建/source/css/_common/components/third-party/gitalk.styl文件，添加内容：

.gt-header a, .gt-comments a, .gt-popup a
  border-bottom: none;
.gt-container .gt-popup .gt-action.is--active:before
  top: 0.7em;

third-party.styl

修改/source/css/_common/components/third-party/third-party.styl，在最后一行上添加内容，引入样式：

@import "gitalk";

以上就是NexT中添加gitalk评论的配置，博客上传到GitHub上后，打开页面进入某一博客内容下，就可看到评论框啦，使用自己的GitHub第一次登录创建Issues可能会有点慢。

问题

如果评论框没有出现，应该是哪里配置没有填对，也可能是这个原因
